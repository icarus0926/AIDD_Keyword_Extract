Tutorial para anlises de genomas

Autores: MsC. Kelly Hidalgo MsC. Luis Gabriel Cueva Yesqun

Pipeline para montagem e anotao taxonmica e funcional de genomas de isolados. Para este tutorial ser usado como modelo um genoma bacteriano,sequenciado na plataforma Illumina (2 x 250 bp). As anlises sero conduzidas seguindo duas abordagens, baseados em linha de comando (CLI Command Line Interface ) e usando ferramentas online (GUI Graphical User Interface ).

WORKFLOW

CLI

0. Organizao de Dados

0.1. Login no servidor

Para os usurios baseados em Windows necessrio a instalao do software MobaXterm .

Aps instalado, clique na opo session , escolha o icone ssh e preencha com os dados do login: remote host (endereo IP). Depois selecione a opo specify name e preencha com o nome do usuario. Por ltimo ser solicitada a senha. Voc pode salvar os dados do login.

0.2. Sequncias

As sequncias foram obtidas do sequenciamento na plataforma Illumina usando o protocolo paired-end (2 x 250 bp) de cepa bacteriana EP216.

Arquivos

  • 216_R1.fastq
  • 216_R2.fastq

Para descarregar os arquivos R1 e R2 .

Dica: Usualmente os arquivos so entregues com nomes codificados do equipamento usado para o sequenciamento que no so informativos para o usurio. Sempre renomee os arquivos com palavras de fcil identificao. Evite espaos e caracteres especiais. Para separar palavras prefira underline _ .

0.2. Criao de diretrios

Os seguintes comandos so para criar, organizar e explorar diretrios.

  • Crie um diretrio base para todo o processo usando o comando mkdir ( make dir ) 
mkdir genomica
  • Entre ao novo diretrio usando o comando cd ( change directory ) 
cd genomica/
  • Crie um novo diretrio para armazenar as sequncias brutas.

Dica: Dado que a maiora das etapas do workflow so sequenciais, recomendvel nomear os diretrios comeando com um nmero e assim manter a organizao. 

mkdir 00.RawData
  • Crie os demais diretrios em uma linha de comando s 
mkdir 01.FastqcReports 02.CleanData 03.Assembly 04.QualityAssembly 05.TaxonomyAnnotation 06.GenePrediction 07.FunctionalAnnotation
  • Use o comando ll ( list ) para listar o contedo do diretrio atual 
ll

0.3. Upload de sequncias

As sequncias devem ser descarregadas da plataforma da facility contratada e subidas ao servidor usando o software MobaXterm. Coloque os arquivos no diretrio 00.RawData

Dica: Para aprender mais comandos teis use o seguinte tutorial de UNIX

1. Controle de Qualidade

1.1. Checagem da qualidade

Nesta etapa ser acessada qualidade das sequncias usando a ferramenta FastQC .

Antes de rodar a anlise, voc deve saber que nos servidores as ferramentas so instaladas usando um gerenciador de programas, chamado Anaconda. O qual facilita a instalao das ferramentas em ambientes virtuais separados, isto porque elas so desenvolvidas en diferentes linguagens de programao (i.e.python, perl, R, etc.). Por tanto precisa ser ativado o ambiente virtual onde foi instalado o FastQC, o qual quality

Dica: Para saber que ferramentas esto instaladas e em que ambiente virtual revise o documento /home/bioinfo/Documents/ferramentas.txt

  • Ative o ambiente quality 
source /home/anaconda3/bin/activate quality

Veja que antes do nome do usurio no prompt deve aparecer o nome do ambiente entre parntese.

  • Execute o comando do FastQC 
fastqc -t 10 00.RawData/* -o 01.FastqcReports/

Sintaxe 

fastqc -t <num ncleos> <input_directory> -o <output_directory>

Dica: TODAS as ferramentas bioinformticas tem um manual de ajuda que pode ser acessado pela linha de comando, usando o flag -h ou --help . ANTES de executar uma ferramenta sempre leia o manual . Este comando lista todos os argumentos disponveis para a ferramenta e explica como devem/podem ser usados.

Ateno: Para o uso correto e seguro do servidor verifique o nmero de ncleos disponveis para o usurio no momento da anlise. NUNCA trabalhe com o total dos ncleos da mquina.

Os arquivos .html gerados no diretrio 01.FastqcReports podem ser baixados usando os recursos grficos de MobaXterm.

Para maiores detalhes sobre a interpretao e anlise dos relatrios do FastQC consulte aqui .

1.2. Trimagem e filtragem de sequncias

A remoo de sequncias e bases de baixa qualidade deve ser feita baseada nos relatrios da checagem da qualidade. O programa Trimmomatic , atravs de diversos argumentos realiza a filtragem e trimagem das sequncias.

  • Segundo os resultados obtidos da qualidade execute o seguinte comando do Trimmomatic: 
trimmomatic PE -threads 10 00.RawData/216_R1.fastq 00.RawData/216_R2.fastq 02.CleanData/216_R1.pair.fastq 02.CleanData/216_R1.unpair.fastq 02.CleanData/216_R2.pair.fastq 02.CleanData/216_R2.unpair.fastq LEADING:3 TRAILING:3 CROP:249 HEADCROP:15 SLIDINGWINDOW:4:15 MINLEN:100

Sintaxe 

trimmomatic PE -threads <num_nucleos> <file_input_R1> <file_input_R2> <file_output_pair_R1> <file_output_unpair_R1> <file_output_pair_R2> <file_output_unpair_R2> [argumentos]

Consulte o manual da ferramenta para obter mais detalhes.

Verifique que a etapa de trimagem e filtragem foi bem sucedida executando FastQC para as sequncias pareadas ( 02.CleanData/216_R1.pair.fastq e 02.CleanData/216_R2.pair.fastq ).

2. Montagem de novo

2.1. Spades

Uma montagem o processo de alinhamento de sequncias curtas com o objetivo de recuperar uma sequncia maior. No caso da montagem de novo so usados algortmos baseados em k-mers , que so subsequncias com tamanho definido pelo usurio (i.e.21-mers).

O programa Spades est instalado no ambiente assembly , por tanto deve ser ativado com o seguinte comando 

source /home/anaconda3/bin/activate assembly

A montagem ser realizada executando o comando:

Dica: O program nohup permite executar tarefas em segundo plano, com o objetivo de manter a execuo do comando ainda com a perda da conexo. 

nohup spades.py --careful -1 02.CleanData/216_R1.pair.fastq -2 02.CleanData/216_R2.pair.fastq -k 21,33,55,77,99,111,127 -o 03.Assembly/ -t 10

Sintaxe 

spades.py -1 <pair1> -2 <pair2> -k <kmers list> -o <output_directory> -t <num_nucleos>

O flag --careful tenta reduzir o nmero de mismatches e indels curtos. recomendado para montagem de genomas de procariotas.

No manual encontra mais detalhes.

2.2. Avaliao da montagem

A montagem precisa ser avaliada atravs de mtricas que representam qualidade do genoma. Neste paso ser calculado o N50, o nmero de contigs, o tamanho do genoma, a completude e o nvel de contaminao.

Primeiro ser executado o progama Quast o qual est instalado no ambiente quast . 

source /home/anaconda3/bin/activate quast

Para gerar o relatrio com as mtricas use o seguinte comando: 

quast 03.Assembly/scaffolds.fasta -o 04.QualityAssembly/ --threads 10

Sintaxe 

quast <input_assembly> -o <output_directory> -t <num_nucleos> [opes]

Descarregue o arquivo 04.QualityAssembly/report.html onde poder explorar todas as mtricas de qualidade.

Dica: Esta ferramenta permite comparar diversas montagens das mesmas sequncias obtidas de diferentes montadores e/ou executados com diferentes parmetros. Esta estratega permite escolher a melhor montagem. Para maiores detalhes, visete o manual

Contigs com tamanho menor de 600 bp no tem um valor representativo na montagem, recomendvel filtrar essas sequncias.

O programa bbmap tem um script que permite realizar a filtragem por tamanho. bbmap est instalado no ambiente bioinfo: 

source /home/anaconda3/bin/activate bioinfo

reformat.sh in=03.Assembly/scaffolds.fasta out=03.Assembly/216_scaffolds_filtered.fasta minlength=600

Para avaliar a completude e a contaminao da montagem usada a ferramenta CheckM . A qual usa uma base de dados prpria de genes ortologos de cpia nica.

Ative o ambiente checkm 

source /home/anaconda3/bin/activate checkm

Execute a nalise para a montagem completa e filtrada com o seguinte comando: 

checkm lineage_wf 03.Assembly/ 04.QualityAssembly/ -t 10 -x fasta --tab > 04.QualityAssembly/output.txt

Sintaxe 

checkm lineage_wf <input_directory/> <output_directory/> -t <num_nucleos> -x <format> --tab > output.txt

Explore o arquivo de sada 04.QualityAssembly/output.txt usando o comando less . Descarregue o relatrio no seu computador.

Para mais detalhes sobre a interpretao do relatrio visite este link.

3. Anotao Taxonmica

A classificao baseada em dados de genoma tem maior poder ressolutivo em comparao ao usar apenas um gene marcado, desde que esta abordagem analisa mltiplos genes que refletem de forma mais robusta a relaes de parntesco do organismo de interesse. O GTDB-tk uma ferramenta que identifica 120 genes marcadores e os compara com uma base de dados curada e constamente atualizada.

Esta ferramenta est instalada no ambiente gtdbtk 

source /home/anaconda3/bin/activate gtdbtk

Execute o seguinte comando: 

gtdbtk classify_wf --genome_dir 03.Assembly --out_dir 05.TaxonomyAnnotation/ -x fasta --cpus 10

Sintaxe 

gtdbtk classify_wf --genome_dir <input_directory/> --out_dir <output_directory/> -x <format> --cpus <num_nucleos>

O programa gera vrios arquivos de sada que esto resumidos no 05.TaxonomyAnnotation/gtdbtk.bac120.summary.tsv .

4. Anotao Funcional

4.1. Predio de ORFs ( Open Reading Frames )

O objetivo desta etapa procurar as ORFs dentro dos contig/scaffols. Ou seja, predizer onde iniciam e terminam os genes. Basicamente o programa procura por codons de inicio e de parada. Para este objetivo ser usada a ferramenta Prodigal .

Primeiro o ambiente annotation ser ativado 

source /home/anaconda3/bin/activate annotation

Para rodar o program utilize o seguinte comando: 

prodigal -i 03.Assembly/216_scaffolds_filtered.fasta -o 06.GenePrediction/216_scaffolds_filtered_orf -a 06.GenePrediction/216_scaffolds_filtered_proteins.faa -d 06.GenePrediction/216_scaffolds_filtered_ORFnucleotides.fa -s 06.GenePrediction/216_scaffolds_filtered_genes

Sintaxe 

prodigal -i <input_assembly.fasta> -o <output_gbk_file> -a <output_protein_seq_file> -d <output_nucleotides_seq_file> -s <output_coordinates_file>

O arquivo 06.GenePrediction/216_scaffolds_filtered_orf est escrito em formato genbank incluindo as sequncias codificadoras e no codificadoras e as posies delas em cada contig. Adicionalmente, o programa ger arquivos com as sequncias nucleotdicas ( 06.GenePrediction/216_scaffolds_filtered_ORFnucleotides.faa ) e protecas ( 06.GenePrediction/216_scaffolds_filtered_proteins.fa ) de cada gene predito.

4.2. Atribuio Funcional das ORFs

Nesta etapa as ORFs preditas sero funcionalmente anotadas por homologia, usando bases de dados e um alinhador. Podero ser usadas as sequncias nucleotdicas ou protecas.

Para o alinhamento ser usado o programa Diamond , e uma base de dados acurada como KEGG .

Lembre-se de ter ativado o ambiente que contm o Diamond ( annotation ).

Use o seguinte comando para rodar Diamond: 

nohup diamond blastx --threads 10 --more-sensitive -k 1 -f 6 qseqid qlen sseqid sallseqid slen qstart qend sstart send evalue bitscore score length pident qcovhsp --id 60 --query-cover 60 --db /home/bioinfo/Documents/databases/diamond/keggdb.dmnd --query 06.GenePrediction/216_scaffolds_filtered_ORFnucleotides.fa -o 07.FunctionalAnnotation/216_scaffolds_filtered_keggdb.txt --tmpdir /dev/shm &

O arquivo de output uma tabela que contm os cdigos dos genes e as anotaes (nmero KEGG), alm de outras informaes relacionadas ao alinhamento de cada sequncia (i.e.cobertura, % de identidade). Devido a que os genes foram anotados apenas com os nmeros KEGG, necessrio cruzar esta informao com as anotaes detalhadas para cada nmero KEGG, que indicam a funo e categoria dos genes. A continuao encontrar uma srie de passos para obter a tabela final de anotao funcional:

  1. Separar a coluna dos cdigos KEGG 
perl -pe 's/\|?(?:\s+gi|ref)?\|\s*/\t/g' 07.FunctionalAnnotation/216_scaffolds_filtered_keggdb.txt > 07.FunctionalAnnotation/216_scaffolds_filtered_keggdb_formated.txt
  1. Cortar as colunas 1 (Contig ID) e 4 (KEGG ID) e gravar em um novo arquivo 
cut -f1,4 07.FunctionalAnnotation/216_scaffolds_filtered_keggdb_formated.txt > 07.FunctionalAnnotation/216_ID_contigs_keggs.tsv
  1. Ordenando 
sort -k2,2 07.FunctionalAnnotation/216_ID_contigs_keggs.tsv > 07.FunctionalAnnotation/216_ID_contigs_keggs_sorted.tsv
  1. Assignao randmica 
cat 07.FunctionalAnnotation/216_ID_contigs_keggs_sorted.tsv | while read line ; do echo "$line" | join -1 2 -2 1 -e"NA" -o1.1,0,2.2,2.3,2.4,2.5 -t $'\t' - /home/bioinfo/Documents/databases/diamond/kegg_sort.tsv | shuf -n1 >> 07.FunctionalAnnotation/216_keggs_randomic_analysis.tsv ; done

Descarregue e explore a tabela final de anotaes 07.FunctionalAnnotation/216_keggs_randomic_analysis.tsv .

GUI

Existem diversas plataformas online nas quais podem ser realizadas diversas anlises bioinformticas. As mais conhecidas so KBase e Galaxy . Neste tutorial ser usado o KBase, alm da plataforma de montagem e anotao do NCBI .

0. Organizao de Dados

0.1. Criando uma conta no KBase

Ingresse no link para criar uma conta: singup!

Neste link voc pode visitar o extenso manual/tutorial da plataforma.

Para facilitar acesse atravs de uma conta Google

No KBase, voc pode crias fluxos de trabalho compartilhveis e reproduzveis chamados Narrativas que incluem dados, etapas de anlise, resultados, visualizaes e comentrios.

0.2. Upload dos dados

O primeiro passo criar uma nova narrativa clicando em + New Narrative . Uma vez criada a nova narrativa, troque o nome dela para Genmica, clicando no ttulo acima a esquerda Untitled

Na seo DATA ser aberta uma aba para carregar os dados desde uma pasta no seu computador. Selecione os arquivos 216_R1.fastq e 216_R2.fastq

Aps subidos os arquivos na plataforma, eles devem ser importadas para dentro da narrativa, para isto necessrio selecionar o tipo de dados. Para o caso das sequncias deste tutorial, trata-se do formato FASTQ Reads Noninterleaved , clique em Import selected .

Automaticamente ser aberta a primeira ferramenta a ser usada Import from Staging Area . Preencha os campos conforme a foto e clique em Run .

A ferramenta primeiro far a leitura dos arquivos e depois ir juntar eles num objeto s ( genome216 ) e ficaram disponveis dentro da narrativa na aba a esquerda DATA . As ferramentas a serem usadas podem ser procuradas na aba APPS a esquerda, na opo de busca.

1. Controle de Qualidade

1.1. Checagem da qualidade

Nesta etapa ser acessada qualidade das sequncias usando a ferramenta FastQC . Procure no catlogo e clique no nome. Imediatamente ser inserida na narrativa logo aps a ferramenta de importao.

Selecione o objeto genome216 no input ( Read Library/RNA-seq Sample Set )

As anlises no KBase podem demorar um pouco mais do normal de um servidor, devido que se trata de un servidor pblico usado por muitas pessoas ao mesmo tempo.

Uma vez a anlise seja concluida, voc ter acesso aos resultados na aba Results . Na seo Report sero disponibilizados os relatrios FastQC. Se voc quiser descarreg-los na seo Files se encontram os arquivos para cada pair.

1.2. Trimagem e filtragem de sequncias

A remoo de sequncias e bases de baixa qualidade deve ser feita baseada nos relatrios da checagem da qualidade. O programa Trimmomatic , atravs de diversos argumentos realiza a filtragem e trimagem das sequncias. Procure pela ferramenta na barra de busca e adicione ela na narrativa.

No input, selecione o objeto genome216 . Clique em show advanced para modificar os argumentos de corte e filtragem. Segundo os resultados obtidos no relatrio de qualidade, insira os parmetros como apresentado na imagem:

A anlise ter terminado quando mostrar a palavra Sucess . Explore os resultados clicando na aba Results

Perceba que na seo DATA , agora devem aparecer novos objetos criados aps a trimagem. Esses novos elementos so as sequncias limpas pareadas ( cleandata_paired ), e as sequncias no pareadas forward ( cleandata_unpaired_fwd ) and reverse ( cleandata_unpaired_rev ).

Aps a limpeza das sequncias recomendvel repetir a anlise com Fastqc para ver a qualidade das sequncias filtradas pareadas ( cleandata_paired ).

2. Montagem de novo

2.1. Spades

Uma montagem o processo de alinhamento de sequncias curtas com o objetivo de recuperar uma sequncia maior. No caso da montagem de novo so usados algortmos baseados em k-mers , que so subsequncias com tamanho definido pelo usurio (i.e.21-mers).

Procure o montador Spades na barra de busca na seo APPS e coloque ele dentro da narrativa para ter acesso as funes dele.

Clique em show advance para modificar os parmetros da montagem:

Ao rodar Spades dentro do KBase, uma vez terminada a montagem, automaticamente ser rodado o program Quast para analisar a qualidade. Voc pode ter acesso ao relatrio de sada do Quast na aba Results .

Descarregue o relatrio na seo Links , clicando no link QUAST report onde poder explorar todas as mtricas de qualidade.

A montagem ( SPAdes.Assembly ) deve aparecer na aba DATA .

2.2. Skesa

Os desenvolvedores do NCBI criaram a plataforma automatizada RAPT de montagem e anotao funcional. O montador a ferramenta Skesa . A anotao funcional segue o pipeline PGAP , o qual usa como base de dados RefSeq proteins.

Para realizar a montagem se deve submeter os arquivos R1 e R2 filtrados por qualidade, obtidos do KBase. Acesse plataform aqui . Lembre-se que deve se cadastrar com uma conta do NCBI.

O processo demorar aproximadamente 2 horas e os resultados ficaram disponveis na nuvem durante 6 semanas.

Sete arquivos sero gerados e deveram ser descarregados no computador. Este arquivos incluem:

  • skesa_out.fa : montagem em formato fasta
  • assembly_stat_report.tsv : relatrio da qualidade da montagem (nmero de contigs, N50, etc)
  • ani-tax-report.txt : relatrio de identificao baseado em ANI.
  • annot. *: Anotao funcional dos genes preditos em diferentes formatos.

2.3. Avaliao da montagem

Contigs com tamanho menor de 600 bp no tem um valor representativo na montagem, recomendvel filtrar essas sequncias. Dentro do KBase existe uma ferramenta chamada Filter Assembled Contigs by Length que pode ser usada para este objetivo.

Ser criado o novo objeto, chamado assembly_filtered

Se voc filtrou a montagem, daqui para frente voc tem duas montagens. Nos passos seguintes avalie as duas montagens para ter certeza que a filtragem no impactou na qualidade da completude e da anotao de genes.

Para avaliar a completude e a contaminao da montagem usada a ferramenta CheckM . A qual usa uma base de dados prpria de genes ortologos de cpia nica.

Dica: Se tiver mais de um assembly, voc pode criar um AssemblySet para colocar todos dentro de um objeto s, e assim facilitar as anlises, podendo fazer vrios elementos ao mesmo tempo. Basta procurar na barra de buscas de APPS Build AssemblySet .

Procure o CheckM no catlogo de ferramentas e coloque ele dentro da narrativa. Nas opes avanadas, selecione full tree no Reference Tree parameter, e save no Save All Plots . Uma vez finalizada a anlise, na aba Results , encontrar um grfico apresentando o resultado. Clicando no link CheckM Table , poder obter as informaes de Completude e Contamination, alm da linagem marcadara usada para a anlise.

Na seo Files , se encontram os arquivos de sada gerados para serem descarregados.

3. Anotao Taxonmica

A classificao baseada em dados de genoma tem maior poder ressolutivo em comparao ao usar apenas um gene marcado, desde que esta abordagem analisa mltiplos genes que refletem de forma mais robusta a relaes de parntesco do organismo de interesse. O GTDB-Tk uma ferramenta que identifica 120 genes marcadores e os compara com uma base de dados curada e constamente atualizada.

Coloque a ferramenta GTDB-Tk dentro da narrativa, e clique em show advanced :

O resultado ser uma tabela com a classificao taxonmica por vrias abordagens.

A identificao taxonmica com dados genmicos pode ser realizada com base nas mtricas OGRI, que incluem o ANI e dDDH. Para calcular estas mtricas, submeta a montagem no seguintes sites: * JSpecies * Genome to genome distance calculator

Ao calcular estas mtricas, deve ser incluidos os genomas de referncia das linhagens tipo mais prximas filogenticamente.

4. Anotao Funcional

Dentro do KBase tem vrias opes para fazer a assignao funcional, tais como RASTtk e Prokka. O dois faro primeiro a predio das ORFs e depois a anotao.

RASTtk

Procure no catlogo e escolha Annotate Genome/Assembly with RASTtk e insira na narrativa.

O resultado pode ser acessado na aba Results . Al encontrar varias sees.

Em Objects estar o novo objeto criado AssemblyAnnotation , clicando nele abrir na narrativa uma tabela interativa onde pode explorar e procurar genes, alm de uma aba para explorar os contigs.

Na seo Report tem tambm uma tabela interativa menos detalhada com os genes anotados. Summary apresenta um resumo da anlise, com nmeros de genes encontrados, no codificantes, etc.

Prokka

Em construo

eggNOG Mapper

Esta ferramenta permite realizar a anotao funcional de sequncias genmicas ou de um conjunto de genes preditos em formato fasta. Nas opes avanadas, a plataforma permite alterar vrios parmetros como o tipo de base de dados, o alinhador, a cobertura e o porcentagem de identidade, etc. Ao submeter a montagem, deve-se preencher o campo Email address para receber uma mensagem com um link que inicia o processo.

Quando o processo acabar, voc pode explorar as anotaes na parte inferior do site. A busca pode ser feita baseado em nome ou cdigo do gene. Alternativamente, pode descarregar a tabela de anotaes em diferentes formatos (csv, excel, gff, etc)

Concluses

A pesar que as ferramentas apresentadas neste tutorial so executadas baseadas em um pipeline, elas se complementan e podem ser usadas sem seguir uma sequncia.

Imprevistos ou periodos de manuteno podem interromper a disponibilidade das ferramentas online. De tal forma que estar familiarizado com a linha de comando deve ser uma prioridade para o usurio.
